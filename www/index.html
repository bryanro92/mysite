About this Site.

Technical Stuff.

Google Cloud Run.

This site scales to zero when no one is using it! That is one advantage to Cloud Run's serverless computing platform. I like to reduce my bills, and I could help you reduce yours as well while optimizing for reliability.

Infrastructure

What happens when you visit the site, but the application wasn't deployed and web server was not running?

  • As DNS resolves in the users browser, their HTTP GET request to my site connects to a fully managed Google Load Balancer which terminates the TLS connection and triggers the deployment of my application.
  • More specifically, the trigger creates a k8s deployment on a google managed cluster and the container is deployed, this is known as a cold start.
  • A scheduler determines a virtual machine to deploy my container to, and the container is downloaded from my personal private docker registry.
  • My web server starts up, within milliseconds, and is ready to begin serving requests.
  • Once the client connections are terminated, the application scales to zero.

Scale to Zero

  • For a short amount of time the container is placed into an idle state, but unable to access any processor resources, and therefore I am no longer incurring any charges.
  • If a connection comes in during this idle state, the cpu lock is removed and the container resumes serving requests.
  • If no client requests come in the next few minutes then the container is shutdown and will proceed with a cold start next time the site is accessed.

Security

  • While the client has established an encrypted TLS connection with the front end load balancer, the load balancer back end encapsulates the HTTP request and crafts a RPC layer 7 request which is protected using Application Layer Transport Security (ALTS).
  • ALTS was created by Google in 2007, and is managed and maintained by Google.
  • Since ALTS runs on Google hardware they are able to easily upgrade and maintain it allowing them to quickly define what cryptographic protocols are in use.
  • This ensures state of the art encryption and reduces my operational responsibility in mitigating or defending against zero days.

Source Code

  • My site was developed to take advantage of how Cloud Run works. Cloud Run charges for compute time which increases per HTTP request, and I've built this as a single page app.
  • After the original request of my index.html, your browser requests the rest of my page resources such as images, css, and javascript files.
  • I leveraged scalable vector graphics (SVG) icons where possible to reduce the number of HTTP requests required for the site to load.
  • Under the hood this is a stateless site built purely on HTML and javascript, with additional javascript libraries like jquery.
  • You can navigate around the site without sending additional requests to Cloud Run. Stay long enough and my app will shutdown and scale to zero while you can click around and consider how my knowledge, skillset, and experience could help your business.
  • I use the continuous deployment integration of Cloud Run that allows pushing to the source repo to automatically trigger the build and deployment of the new version.
