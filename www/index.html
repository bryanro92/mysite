About this Site.

This site scales to zero when no one is using it! I like to reduce my bills, and I could help you reduce yours as well while optimizing for reliability.

More specifically, when the first client connection is established the container is deployed, this is known as a cold start. My web server starts up, within milliseconds, and is ready to begin serving requests. Once the client connections are terminated, the application scales to zero.

Scale to Zero

For a short amount of time the container is placed into an idle state, but unable to access any processor resources, and therefore I am no longer incurring any charges.

If a connection comes in during this idle state, the cpu lock is removed and the container resumes serving requests.

If no client requests come in the next few minutes then the container is shutdown and will proceed with a cold start next time the site is accessed.
